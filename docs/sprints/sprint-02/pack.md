# NEX Sprint 02 Pack

## Goal

Make NEX stateful, recoverable, and proof-aware on top of the Sprint 01 execution spine.

## Deliverables

- events
- state projections
- resume packets
- pattern records
- evidence
- verifications
- gate decisions
- audit records
- snapshots
- memory timeline route
- proof vault route
- gates and blockers route

## Closure Conditions

1. Events persist and can be queried by scope.
2. Current state can be read from projections.
3. Resume packets identify blockers and the next required action.
4. Evidence attaches to tasks and runs.
5. Blocking verifications prevent task closure.
6. Audit records and snapshots are queryable by scope.
